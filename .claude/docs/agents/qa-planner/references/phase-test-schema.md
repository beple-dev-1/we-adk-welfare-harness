# 페이즈별 테스트 계획 형식

각 개발 페이즈에 대응하는 TC 목록 형식이다.

## TC 번호 규칙
```
TC-{과업번호}-{도메인코드}-{순번 3자리}
```

도메인 코드:
- `BENEFIT` — 복지혜택 (지급, 취소, 조회)
- `MEMBER` — 회원 (등록, 수정, 조회, 탈퇴)
- `MERCHANT` — 가맹점 (등록, 조회, 상태관리)
- `SETTLEMENT` — 정산 (집계, 배치)
- `COMMON` — 공통 (인증, 예외처리, 유틸)

## TC 항목 형식
```markdown
### TC-{번호}: {제목}

| 항목 | 내용 |
|------|------|
| 유형 | 단위 / 슬라이스 / 통합 |
| 우선순위 | 높음 / 중간 / 낮음 |
| 전제조건 | {테스트 실행 전 필요한 상태} |
| 입력 | {테스트 입력값} |
| 기대결과 | {예상 동작 및 응답} |
| 대응 개발 항목 | phase_{N}.md의 구현 항목 |
```
