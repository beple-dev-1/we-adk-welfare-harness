# /pack 스킬

세션 종료 전 작업 상태와 이력을 기록하여 다음 세션 또는 다른 작업자가 이어받을 수 있도록 한다.

## 사용법
```
/pack
```

## 절차

### 1단계: 작업 범위 감지
- 현재 `/develop` 스코프와 과업번호를 확인한다.
- `git status`와 `git log --oneline -10`으로 변경 사항을 파악한다.

### 2단계: HANDOFF.md 갱신
`HANDOFF.md` (루트)를 다음 구조로 갱신한다:

```markdown
# HANDOFF — {날짜}

## 진행 중인 작업
- 과업번호: {N}
- 스코프: {scope}
- 브랜치: {branch}
- 현재 단계: {phase}

## 작업 요약
{이번 세션에서 수행한 작업 요약}

## 변경 파일
{git status/diff 기반 변경 파일 목록}

## 다음 단계
{남은 작업 항목}

## 미결 사항
{해결 안 된 질문이나 결정 사항}

## 보안 자가점검
- [ ] 운영 설정 파일 변경 없음
- [ ] 시크릿 정보 코드 포함 없음
- [ ] 개인정보 로그 노출 없음
```

### 3단계: HANDOFF_HISTORY.md 누적
`HANDOFF_HISTORY.md`에 완료된 작업을 날짜·과업번호 기준으로 추가한다.
완료 작업은 HANDOFF.md에서 HISTORY로 이동한다.

### 4단계: 보안 자가점검
- 시크릿 정보가 코드에 포함되었는지 확인한다.
- 운영 설정 파일이 변경되지 않았는지 확인한다.

## 완료 후
HANDOFF.md 갱신 완료를 안내하고 세션을 종료할 준비가 되었음을 알린다.
