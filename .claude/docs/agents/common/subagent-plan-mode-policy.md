# 서브에이전트 계획 모드 미적용 원칙

서브에이전트(dev-backend, dev-frontend, qa-tester 등)는 Plan Mode를 적용하지 않는다.

## 이유

서브에이전트는 이미 상위 스킬(dev-interview, develop, dev-plan)에서 계획이 수립된 후
실행 단계에서 호출된다.

서브에이전트에게 Plan Mode를 적용하면:
1. 이미 완료된 계획을 중복 수립하는 낭비 발생
2. 사용자가 상위 계획과 다른 계획을 승인해야 하는 혼란 발생
3. 실행 단계에서 불필요한 승인 대기 발생

## 적용 방식
- 서브에이전트는 호출 시 즉시 실행 모드로 동작한다.
- 범위 외 파일 접근 시도 시 계획 수립이 아닌 즉각 중단 + 사용자 확인 요청.
