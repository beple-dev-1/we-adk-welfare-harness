# 구현 시작 전 작업 범위 확인 게이트

구현을 시작하기 전에 아래 케이스 중 해당하는 것을 확인한다.

## 케이스 분류

### 케이스 A — 계획서 있음
- `target/plans/{과업번호}/{과업번호}_dev_plan.md` 파일이 존재한다.
- **행동**: 계획서의 현재 페이즈를 확인하고 해당 페이즈의 구현 대상 파일부터 시작한다.

### 케이스 B — 브리프만 있음
- `target/works/{과업번호}_dev_brief.md` 파일이 존재하지만 계획서가 없다.
- **행동**: `/dev-plan {과업번호}`를 실행하여 계획서를 먼저 생성하도록 안내한다.

### 케이스 C — 즉시 구현 요청
- 계획서·브리프 없이 즉시 구현을 요청하는 경우 (단순 수정, 버그 수정 등).
- **행동**: 범위를 명확히 확인 (어떤 파일의 어떤 부분을 수정하는지) 후 구현.
- 단, 복지혜택 지급·취소·정산 로직은 케이스 C 허용하지 않음 → 반드시 계획서 작성 요구.

## 케이스 A/B 생략 조건
- 단순 코드 스타일 수정, 오타 수정, 주석 추가
- 테스트 코드만 추가하는 경우
- 문서 파일만 수정하는 경우
