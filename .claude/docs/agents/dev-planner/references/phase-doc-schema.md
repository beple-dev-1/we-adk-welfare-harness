# 개발 계획서 페이즈 문서 형식

`target/plans/{과업번호}/phases/phase_{N}.md` 파일의 형식이다.

```markdown
# 페이즈 {N}: {제목}

| 항목 | 내용 |
|------|------|
| 페이즈 번호 | {N} |
| 목표 | {이 페이즈에서 달성할 것} |
| 의존 페이즈 | {선행 페이즈 번호, 없으면 없음} |

## 구현 대상 파일
| 파일 경로 | 작업 유형 | 설명 |
|---------|---------|------|
| src/main/java/.../BenefitService.java | 신규 생성 | 혜택 지급 서비스 인터페이스 |

## 상세 구현 가이드
### {파일명}
{구체적인 구현 방향, 메서드 시그니처, 주의사항}

## 완료 기준
- [ ] {확인 항목 1}
- [ ] {확인 항목 2}
- [ ] 단위 테스트 통과

## 다음 페이즈 연결
{다음 페이즈(N+1)에서 이 페이즈 산출물을 어떻게 사용하는지}
```
